# Pathology Ensemble Visualization Tool

This utility creates an interactive, high-fidelity HTML report from the CSV datasets generated by `json_to_csv.py`. It is designed to provide deep clinical and technical insight into how individual pathology models interact with the LLM Judge.

## Purpose
The visualizer transforms raw probability data into a "Visual Audit Trail," allowing researchers to quickly identify where the ensemble logic succeeded, where the Judge "saved" failing models, and where the system collectively failed.

## Requirements
- **Python 3.10+**
- **uv** (Recommended for zero-setup execution)
- Dependencies (handled automatically by `uv`):
  - `pandas`: Data manipulation
  - `jinja2`: HTML templating
  - `bootstrap 5` (via CDN): Responsive UI
  - `DataTables` (via CDN): Interactive sorting/filtering

## Usage

The script uses `uv` metadata to manage its own dependencies, so no `pip install` is required.

### Basic Visualization
```bash
uv run utils/visualize_results.py path/to/dataset.csv
```

### With Pathology Images
If your CSV contains valid `img_path` entries, you can display thumbnails directly in the report:
```bash
uv run utils/visualize_results.py path/to/dataset.csv --images
```

## Design Features

### 1. Status Badges (The "Narrative" Column)
Every row is automatically categorized to show the "story" of that test case:
- **CONSENSUS** (Green): All models and the Judge agreed on the correct answer.
- **SAVED** (Yellow): All models were incorrect, but the **Judge corrected them** to the right answer.
- **RECONCILED** (Blue): Models disagreed, but the Judge successfully chose the correct one.
- **MISSED** (Red): At least one model had the right answer, but the **Judge chose the wrong one**.
- **FAILED** (Gray): Both the models and the Judge were incorrect.

### 2. Probability Sparklines
Instead of just numbers, each model's probability distribution for choices A, B, C, and D is shown as a horizontal **sparkline (mini-bar chart)**. This allows you to see at a glance if a model was "split" between two answers or "highly confident" in its choice.

### 3. Dynamic Model Support
The script automatically detects how many models are in your CSV (e.g., `conch`, `musk`, `chexagent`) and generates the appropriate number of columns. It will not break if you add or remove models from your evaluation pipeline.

### 4. Color-Coded Accuracy
- **Green Cells**: Indicate a correct prediction or choice.
- **Red Cells**: Indicate an incorrect prediction or choice.
- This allows for "heatmap-style" scanning of the table to find clusters of failures.

### 5. Interactive Table
The output HTML uses `DataTables.js`, providing:
- **Instant Search**: Filter rows by status (e.g., "SAVED") or question text.
- **Multi-Column Sorting**: Sort by probability, correctness, or question number.
- **Pagination**: Handles large datasets smoothly.

## How it Works
1. **Data Loading**: The script reads the CSV and identifies model-specific columns.
2. **Heuristic Analysis**: It compares the `Judge Answer`, `Right Answer`, and `{Model}_Correct` columns to determine the row's `Status`.
3. **Template Injection**: Data is injected into a Bootstrap-based HTML template using the `Jinja2` engine.
4. **Output**: A standalone `.html` file is saved in the same directory as the input CSV.
